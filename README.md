<h2><a id="user-content-installation" class="anchor" aria-hidden="true" href="#installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Installation</h2>
<p><strong>This theme requires Jekyll 3.8 so it is compatible with GitHub Pages.</strong></p>
<p>Add this line to your Jekyll site's <code>Gemfile</code>:</p>
<div class="highlight highlight-source-ruby"><pre><span class="pl-en">gem</span> <span class="pl-s">"bulma-clean-theme"</span></pre></div>
<p>And add this line to your Jekyll site's <code>_config.yml</code>:</p>
<div class="highlight highlight-source-yaml"><pre><span class="pl-ent">theme</span>: <span class="pl-s">bulma-clean-theme</span></pre></div>
<p>And then execute:</p>
<pre><code>$ bundle
</code></pre>
<p>Or install it yourself as:</p>
<pre><code>$ gem install bulma-clean-theme
</code></pre>

<h3><a id="user-content-pages" class="anchor" aria-hidden="true" href="#pages"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Pages</h3>
<p>Create your pages as individual markdown files and use the <code>layout: page</code> for normal pages. Set the pages title and subtitle in the front matter and it will appear in the hero.</p>
<h4><a id="user-content-page-hero" class="anchor" aria-hidden="true" href="#page-hero"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Page Hero</h4>
<p><strong>New in 0.2</strong>
Heros can now display a background image if you provide a <code>hero_image: /path/to/image.jpg</code> setting in your page front matter, or in the <a href="https://jekyllrb.com/docs/configuration/front-matter-defaults/" rel="nofollow">defaults</a> in your sites <code>_config.yml</code></p>
<p>You can also set the height of the hero by providing a bulma hero height class in your front matter, such as <code>hero_height: is-fullwidth</code>. If you do not provide this, it will revert to is-medium</p>
<p><strong>New in 0.5.4</strong>
If you would like to add a call to action button in the hero then add <code>hero_link</code> and <code>hero_link_text</code> to the page's front matter.</p>
<p><strong>New in 0.5.7</strong>
If you would like to hide the hero, you can set <code>hide_hero: true</code> in the page's front matter.</p>
<p><strong>New in 0.7.1</strong>
If you would like to darken the hero so the title stands out more, you can set <code>hero_darken: true</code> in the page's front matter. You can overwrite the default background colour by setting the <code>$hero-darken</code> sass variable.</p>
<h4><a id="user-content-table-of-contents" class="anchor" aria-hidden="true" href="#table-of-contents"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Table Of Contents</h4>
<p><strong>New in 0.5.8</strong>
If you want to display a table of contents (toc) then add <code>toc: true</code> to your page's front matter. You can customise the default table of contents title by setting <code>toc_title: My Custom Title</code> in the page's front matter.</p>
<p><strong>New in 0.10.3</strong>
If you would prefer to display the contents in the menubar at the side of the page, then use <code>menubar_toc: true</code> instead of <code>toc: true</code>. This will also override the page's <code>menubar</code> setting.</p>
<h3><a id="user-content-posts" class="anchor" aria-hidden="true" href="#posts"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Posts</h3>
<p>If you want posts, create a <code>_posts</code> directory to store your posts as per normal Jekyll usage, with the <code>layout: post</code>. Next create a <code>blog</code> directory with an index.html file that has <code>layout: blog</code></p>
<p>Set the paginate and the paginate_path up in the <code>_config.yaml</code> to configure the posts per page and the blog pagination path.</p>
<div class="highlight highlight-source-yaml"><pre><span class="pl-ent">paginate</span>: <span class="pl-c1">5</span>
<span class="pl-ent">paginate_path</span>: <span class="pl-s"><span class="pl-pds">"</span>/blog/page:num<span class="pl-pds">"</span></span></pre></div>
<p><strong>New in 0.2</strong> It will now display an image in the blog page if you set <code>image: /path/to/image.jpg</code> in your post's or page's front matter, or in the <a href="https://jekyllrb.com/docs/configuration/front-matter-defaults/" rel="nofollow">defaults</a> in your sites <code>_config.yml</code></p>
<p>You can also set the height of the hero by providing a Bulma hero height class in your front matter, such as <code>hero_height: is-fullwidth</code>. If you do not provide this, it will revert to is-medium</p>
<h4><a id="user-content-post-excerpt-and-summary" class="anchor" aria-hidden="true" href="#post-excerpt-and-summary"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Post Excerpt and Summary</h4>
<p>By default the blog page and the latest posts will use the automatically generated post excerpt. If you would like to override this you can set the summary in the post's front matter.</p>
<div class="highlight highlight-source-yaml"><pre><span class="pl-ent">layout</span>: <span class="pl-s">post</span>
<span class="pl-ent">title</span>: <span class="pl-s">The post title</span>
<span class="pl-ent">summary</span>: <span class="pl-s">|-</span>
<span class="pl-s">    This is the custom summary for the post.</span>
<span class="pl-s"></span>
<span class="pl-s">    It can be in **markdown format** if required.</span></pre></div>
<h4><a id="user-content-social-share-buttons" class="anchor" aria-hidden="true" href="#social-share-buttons"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Social Share Buttons</h4>
<p>Share buttons will be displayed on your posts unless you hide them by adding <code>hide_share_buttons: true</code> to your config file.</p>
<h3><a id="user-content-navigation" class="anchor" aria-hidden="true" href="#navigation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Navigation</h3>
<p>For the top navigation, create a navigation.yml file in <code>_data</code> directory with the following format with the pages you want to include in the top navigation. You can now also add items to a dropdown menu.</p>
<div class="highlight highlight-source-yaml"><pre>- <span class="pl-ent">name</span>: <span class="pl-s">Page 1</span>
  <span class="pl-ent">link</span>: <span class="pl-s">/page-1/</span>
- <span class="pl-ent">name</span>: <span class="pl-s">Blog</span>
  <span class="pl-ent">link</span>: <span class="pl-s">/blog/</span>
  <span class="pl-ent">dropdown</span>:
    - <span class="pl-ent">name</span>: <span class="pl-s">Page 2</span>
      <span class="pl-ent">link</span>: <span class="pl-s">/page-2/</span></pre></div>
<p>For the current page to have an active class, ensure the <code>link:</code> format matches your <a href="https://jekyllrb.com/docs/permalinks/#extensionless-permalinks" rel="nofollow">permalink</a> format. The above example will work with <code>permalink: pretty</code> setting in your <code>_config.yml</code></p>

<p>To have a fixed navbar, you can set <code>fixed_navbar: top</code> of <code>fixed_navbar: bottom</code> respectively in your

<p>To overwrite the primary theme colour, set a sass variable in <code>assets/css/app.scss</code> before importing <code>main</code></p>
<pre><code>---
---

// Import Main CSS file from theme

</code></pre>

<h3><a id="user-content-menubar" class="anchor" aria-hidden="true" href="#menubar"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Menubar</h3>
<p><strong>New in 0.3</strong></p>
<p>The menubar gets its content from a data file in your site's <code>_data</code> directory. Simply set the name of your data file in the page's menubar setting in the frontmatter.</p>
<div class="highlight highlight-source-yaml"><pre><span class="pl-ent">show_sidebar</span>: <span class="pl-c1">false</span>
<span class="pl-ent">menubar</span>: <span class="pl-s">example_menu</span></pre></div>
<p>You will probably want to disable the show_sidebar otherwise there will be little room for the page content.</p>

          <span class="pl-ent">link</span>: <span class="pl-s">/page-2/</span>

<h4><a id="user-content-creating-a-showcase-datafile" class="anchor" aria-hidden="true" href="#creating-a-showcase-datafile"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Creating A Showcase Datafile</h4>
<p>Create a datafile in your sites <code>_data</code> directory in the following format. Subtitle, features and tags are not required.</p>
<p>The description text accepts markdown and is run through the markdownify filter on the page.</p>
<p>The image_ratio will default to is-16by9 if it is not defined and accepts the <a href="https://bulma.io/documentation/elements/image/" rel="nofollow">Bulma image</a> classes.</p>
<p>To display GitHub Stars, Forks and Watchers badges add your GitHub user and repo name to the github setting, such as <code>chrisrhymes/bulma-clean-theme</code></p>
<p>To change the default styles of the features, set <code>features_styles</code>. This uses the styles from <a href="https://www.csrhymes.com/bulma-block-list/" rel="nofollow">bulma-block-list</a> npm package.</p>
<div class="highlight highlight-source-yaml"><pre><span class="pl-ent">intro</span>: <span class="pl-s">|-</span>
<span class="pl-s">  This is some introduction text for the showcases.</span>
<span class="pl-s"></span>
<span class="pl-s">  ## Example Heading</span>
<span class="pl-s">  It can convert markdown format</span>
<span class="pl-s"></span>
<span class="pl-s"></span><span class="pl-ent">items</span>:
  - <span class="pl-ent">title</span>: <span class="pl-s">Example showcase item</span>
    <span class="pl-ent">subtitle</span>: <span class="pl-s">Example subtitle</span>
    <span class="pl-ent">description</span>: <span class="pl-s">|-</span>
<span class="pl-s">      This is the example description for this item that you are showcasing and has an image, title, description, tags and a link.</span>
<span class="pl-s"></span>    <span class="pl-ent">features</span>:
      - <span class="pl-s">This is a feature</span>
      - <span class="pl-s">This is a feature</span>
    <span class="pl-ent">features_styles</span>: <span class="pl-s">is-centered is-outlined is-primary</span>
    <span class="pl-ent">image</span>: <span class="pl-s">https://via.placeholder.com/1024x788</span>
    <span class="pl-ent">image_ratio</span>: <span class="pl-s">is-16by9</span>
    <span class="pl-ent">link</span>: <span class="pl-s">http://www.example.com</span>
    <span class="pl-ent">link_text</span>: <span class="pl-s">View example</span>
    <span class="pl-ent">tags</span>: <span class="pl-s">PHP,CSS,JavaScript</span>
    <span class="pl-ent">github</span>: <span class="pl-s">user/repo-name</span></pre></div>

<p>The sponsors require a name, but not a profile link.</p>
<h4><a id="user-content-displaying-the-sponsors" class="anchor" aria-hidden="true" href="#displaying-the-sponsors"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Displaying the Sponsors</h4>
<p>To display the sponsors on your page, set the sponsors to the filename without the extension in the page's front matter</p>
<div class="highlight highlight-source-yaml"><pre><span class="pl-ent">layout</span>: <span class="pl-s">page</span>
<span class="pl-ent">title</span>: <span class="pl-s">My Sponsors Page</span>
<span class="pl-ent">sponsors</span>: <span class="pl-s">my_sponsors</span></pre></div>
<h3><a id="user-content-gallery" class="anchor" aria-hidden="true" href="#gallery"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Gallery</h3>
<p>Display a simple image gallery on a page, with image descriptions.</p>
<h4><a id="user-content-create-an-image-gallery-data-file" class="anchor" aria-hidden="true" href="#create-an-image-gallery-data-file"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Create an image gallery data file</h4>
<p>Start by creating a gallery data file using the below format, for example <code>my_gallery.yml</code>.</p>
<div class="highlight highlight-source-yaml"><pre>- <span class="pl-ent">title</span>: <span class="pl-s">Image Gallery Title</span>
  <span class="pl-ent">images</span>:
    - <span class="pl-ent">link</span>: <span class="pl-s">https://via.placeholder.com/800x450</span>
      <span class="pl-ent">large_link</span>: <span class="pl-s">https://via.placeholder.com/1200x675</span>
      <span class="pl-ent">alt</span>: <span class="pl-s">The alt text for the image</span>
      <span class="pl-ent">description</span>: <span class="pl-s">|-</span>
<span class="pl-s">        The image description can be written in **markdown** if required</span>
<span class="pl-s"></span>      <span class="pl-ent">ratio</span>: <span class="pl-s">is-16by9</span>
    - <span class="pl-ent">link</span>: <span class="pl-s">https://via.placeholder.com/800x600</span>
      <span class="pl-ent">large_link</span>: <span class="pl-s">https://via.placeholder.com/1200x675</span>
      <span class="pl-ent">alt</span>: <span class="pl-s">The alt text for the image</span>
      <span class="pl-ent">description</span>: <span class="pl-s">The image description</span>
      <span class="pl-ent">ratio</span>: <span class="pl-s">is-4by3</span></pre></div>
<ul>
<li>If a ratio is not provided it will default to 16 by 9. Use <a href="https://bulma.io/documentation/elements/image/" rel="nofollow">Bulma image</a> classes to define the image ratio required.</li>
<li>The description can be plain text or it can be markdown if required.</li>
<li>The alt will be used as the images alt text.</li>
<li>The large_link displays in a modal when the image is clicked</li>
</ul>
<h4><a id="user-content-displaying-the-gallery" class="anchor" aria-hidden="true" href="#displaying-the-gallery"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Displaying the gallery</h4>
<p>In your pages front matter add a gallery with the datafiles filename without the extension.</p>
<div class="highlight highlight-source-yaml"><pre><span class="pl-ent">layout</span>: <span class="pl-s">page</span>
<span class="pl-ent">title</span>: <span class="pl-s">My Image Gallery</span>
<span class="pl-ent">gallery</span>: <span class="pl-s">my_gallery</span></pre></div>
<h3><a id="user-content-disqus" class="anchor" aria-hidden="true" href="#disqus"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Disqus</h3>
<p>Disqus comments are available for posts. To be able to use them, you need to set your disqus shortname in <code>_config.yml</code>. Then you need to set your Jekyll environment to production:</p>
<p><code>JEKYLL_ENV=production bundle exec jekyll build</code>.</p>
<p>Comments are enabled by default. If you want to disable them, set in the front matter this setting:</p>
<div class="highlight highlight-source-gfm"><pre>comments: false</pre></div>
